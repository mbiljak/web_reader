# Web Reader

A Firefox extension that extracts the readable text from any webpage and reads it
aloud using **premium macOS voices** (via the `say` command). Includes RSVP
"speed caption" mode (one word at a time with the focal letter highlighted),
live speed/voice control, text-selection reading, and keyboard shortcuts.

## How it works

```
popup.html/js ──┐                         ┌── macOS `say`
                ├─ background.js ── native ┤   (premium voices,
content.js  ────┘   messaging      host    └    pause/resume)
   │
   └─ extracts text (Readability.js), shows RSVP captions
```

- **content.js** — extracts article text, splits into sentences, drives playback
  and the RSVP caption overlay. Falls back to the browser's built-in Web Speech
  voices if the native host isn't available.
- **background.js** — bridges the page to the native host (content scripts can't
  use native messaging) and routes the keyboard shortcuts. Enforces one reading
  tab at a time.
- **native/web_reader_host.py** — receives sentences and speaks them with `say`.
  Launched via `native/launch.sh` (generated by `install.sh`).

## First-time setup

1. **Register the native host** (enables premium macOS voices):
   ```bash
   ./install.sh
   ```
   This writes a launcher with the absolute path to your `python3` (Firefox's
   launch environment can't see mise/pyenv/homebrew pythons) and registers it at
   `~/Library/Application Support/Mozilla/NativeMessagingHosts/`.

2. **Fully quit Firefox** (⌘Q — not just the window) and reopen it, so it picks
   up the newly registered host.

3. **Load the extension**:
   - Temporary (for development): `about:debugging` → *This Firefox* →
     *Load Temporary Add-on* → select `manifest.json`. Unloads on restart.
   - Permanent: install the signed `.xpi` (see below).

## Usage

- Click the toolbar icon, pick a voice (e.g. **Zoe (Premium)**), press ▶.
- **Select text first** to read only that selection; otherwise the whole article
  is read.
- **Speed captions** toggle shows one word at a time with the focal letter in red.
- Speed and voice can be changed **while playing**.

### Keyboard shortcuts

| Shortcut | Action |
|----------|--------|
| `⌃⇧S` | Play / pause |
| `⌃⇧X` | Stop |
| `⌃⇧R` | Open the popup |

Rebind them in `about:addons` → gear icon → *Manage Extension Shortcuts*.

## Building a permanent signed `.xpi`

Requires a free [addons.mozilla.org](https://addons.mozilla.org) account and API
credentials from <https://addons.mozilla.org/developers/addon/api/key/>.

```bash
export PATH="$HOME/.local/share/mise/installs/node/22/bin:$PATH"  # if node is via mise
npm install                                # one time
export WEB_EXT_API_KEY='user:XXXXXXXX:XXX' # JWT issuer
export WEB_EXT_API_SECRET='...'            # JWT secret
npm run sign
```

The signed `.xpi` is written to `web-ext-artifacts/`. Open it in Firefox (or drag
onto `about:addons`) to install permanently. `--channel=unlisted` means it's
signed for self-distribution, not published to the public add-on store.

Useful scripts:
- `npm run lint` — validate before signing (should report 0 errors).
- `npm run build` — produce an unsigned `.zip` to inspect what gets packaged.

## Making updates

### Changing the extension (popup, content, background, icon)

1. Edit the files.
2. Bump the `version` in **`manifest.json`** — required for every signed release
   (Mozilla rejects re-signing the same version). Use an increasing value, e.g.
   `2.0` → `2.1`. Keep `package.json`'s `version` in sync (optional but tidy).
3. Test as a temporary add-on first: `about:debugging` → *Load Temporary Add-on*,
   or click *Reload* if already loaded. (No Firefox restart needed for JS/HTML
   changes — only for native-host changes.)
4. Re-sign: `npm run sign`. Install the new `.xpi` over the old one — Firefox
   recognizes the matching ID (`web-reader@local`) and updates in place.

### Changing the native host (`native/web_reader_host.py`)

1. Edit the Python.
2. If you only changed the host *logic*, no re-register needed — but you must
   **fully quit and reopen Firefox** (it spawns a fresh host process per session).
3. If you changed the host's *path or filename*, re-run `./install.sh` and restart
   Firefox.
4. Quick host test without Firefox:
   ```bash
   ./install.sh    # regenerates launch.sh
   printf '' | python3 native/web_reader_host.py   # should start without error
   ```

### Changing keyboard shortcuts

Edit the `commands` block in `manifest.json`, bump the version, and reload. Users
can always override via *Manage Extension Shortcuts*.

## Requirements

- macOS (uses the `say` command for premium voices).
- `python3` available (`xcode-select --install` if missing).
- Firefox.
- Node.js (only for signing).
